It is an absolute eyesore. Piling an unreadable mountain of `-D` switches directly onto a single command line makes the file hard to scan and turns debugging into a nightmare.

Thankfully, CMake and GNU Make give us a much cleaner way to handle this. We can pull those ugly compiler definitions out of the inline command completely and organize them as a clean, structured list at the top of the file. Then, we can use Make's native string manipulation functions to format them automatically.

---

## The Elegant, Readable Refactor

Open `infra/make/openvino.mk` and replace the contents with this clean layout. It groups the platform fixes into a dedicated array block:

```makefile
# ==============================================================================
# Filename:    infra/make/openvino.mk
# Purpose:     Intel OpenVINO Inference Acceleration Compilation Blueprint
# Type:        Makefile Component (Dynamic Build-Isolation Compliant)
# Attribution: fekerr & Gemini (20260701_1220 / flash 3.5 extended)
# ==============================================================================

BUILD_DIR     ?= build/openvino_relwithdebinfo
LOG_FILE_PATH ?= $(CURDIR)/$(BUILD_DIR)/logs/build_default.log

# --- CLEAN COMPILER PATCH MATRIX ---
# Missing Khronos OpenCL enums caused by Intel oneAPI header injection collisions
OPENCL_PATCH_DEFS := \
    CL_EXTERNAL_MEMORY_HANDLE_D3D11_TEXTURE_KHR=0x406E \
    CL_EXTERNAL_MEMORY_HANDLE_D3D11_TEXTURE_KMT_KHR=0x406F \
    CL_EXTERNAL_MEMORY_HANDLE_D3D12_HEAP_KHR=0x4070 \
    CL_EXTERNAL_MEMORY_HANDLE_D3D12_RESOURCE_KHR=0x4071

# Use Make's addprefix to automatically prepend "-D" to every token in the array
OPENVINO_CXX_FLAGS := $(addprefix -D,$(OPENCL_PATCH_DEFS))

# --- ENFORCED ENVIRONMENT POINTER INTERPOLATION ---
ifeq ($(OpenVINO_DIR),)
    OPENVINO_SEARCH_PATHS := \
        /usr/lib/cmake/openvino2024.6.0 \
        /opt/intel/openvino \
        /usr/lib/x86_64-linux-gnu/cmake/OpenVINO
    TARGET_OV_DIR := $(firstword $(wildcard $(OPENVINO_SEARCH_PATHS)))
else
    TARGET_OV_DIR := $(OpenVINO_DIR)
endif

.PHONY: build-openvino clean-openvino

build-openvino:
	@echo "[Make] Initializing OpenVINO compilation inside: $(BUILD_DIR)"
	@mkdir -p $(BUILD_DIR) $(dir $(LOG_FILE_PATH))
	@if [ -f "$(LOG_FILE_PATH)" ]; then \
		ARCHIVE_TIME=$$(date +%Y%m%d_%H%M%S); \
		echo "[Make] Archiving historical session to logs/build_default.$$ARCHIVE_TIME.log"; \
		mv "$(LOG_FILE_PATH)" "$(dir $(LOG_FILE_PATH))build_default.$$ARCHIVE_TIME.log"; \
	fi
	@echo "==================================================================" >> $(LOG_FILE_PATH)
	@echo "[Make Session] Launching Build at $$(date)" >> $(LOG_FILE_PATH)
	@echo "==================================================================" >> $(LOG_FILE_PATH)
	@echo "[Make] Log Target Destination: $(LOG_FILE_PATH)"
	@START_TIME=$$(date +%s); \
	cd $(BUILD_DIR) && \
	cmake ../../$(ENGINE_DIR) \
		-DGGML_OPENVINO=ON \
		-DCMAKE_BUILD_TYPE=$(CMAKE_BUILD_TYPE) \
		-DOpenVINO_DIR=$(TARGET_OV_DIR) \
		-DCMAKE_CXX_FLAGS="$(OPENVINO_CXX_FLAGS)" >> $(LOG_FILE_PATH) 2>&1 && \
	$(MAKE) -j$(NUM_BUILD_JOBS) >> $(LOG_FILE_PATH) 2>&1; \
	STATUS=$$?; \
	END_TIME=$$(date +%s); \
	DURATION=$$((END_TIME - START_TIME)); \
	if [ $$STATUS -ne 0 ]; then \
		echo "[!] OpenVINO Compilation Macro Failed. Inspect $(LOG_FILE_PATH)"; \
		exit $$STATUS; \
	fi

clean-openvino:
	@echo "[!] Purging isolated target directory: $(BUILD_DIR)"
	rm -rf $(BUILD_DIR)

```

## How This Works Behind the Scenes

By breaking the problem down into structured variables, we use a classic GNU Make tool:

```makefile
OPENVINO_CXX_FLAGS := $(addprefix -D,$(OPENCL_PATCH_DEFS))

```

During Phase 1 evaluation, Make loops through each item in your multiline `OPENCL_PATCH_DEFS` list and attaches `-D` right to the front of it. When it passes the result down to the `cmake` line, it expands cleanly into a well-spaced, single-line configuration string:

```bash
-DCMAKE_CXX_FLAGS="-DCL_EXTERNAL_MEMORY_HANDLE_D3D11_TEXTURE_KHR=0x406E -DCL_EXTERNAL_MEMORY_HANDLE_D3D11_TEXTURE_KMT_KHR=0x406F ..."

```

This keeps your actual execution block uncluttered and readable, preserves the multi-level timestamped log rotation, and cleanly resolves the compiler issues for both `ggml-openvino-extra.cpp` and `ggml-openvino.cpp`.
