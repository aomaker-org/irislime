## Branch: 20260620 - Intel SYCL Runtime Patch

### The Problem

The Intel Level Zero (`libze_loader`) runtime attempts to force-load an optional diagnostic `libze_tracing_layer.so.1`. In the current environment, this library triggers a `SIGSEGV` segmentation fault during hardware initialization (`zelLoaderTracingLayerInit`), causing the `llama-cli` to crash before the model loads.

### The Workaround: "The Dummy Tracer"

Because the loader fails to respect `ZET_` environment variables on this system, we use a library-shadowing technique.

1. **Create the Dummy Library:**
We provide a "null-op" shared object that satisfies the linker's search for `zelLoaderTracingLayerInit`.
```bash
# Create the source
echo 'void zelLoaderTracingLayerInit() { return; }' > dummy_tracing.c
# Compile to shared object
gcc -shared -fPIC -o libze_tracing_layer.so.1 dummy_tracing.c

```


2. **Implementation:**
We shadow the system library by placing our version in a custom path and prepending that path to `LD_LIBRARY_PATH`.

### Environment Configuration (`setup_sycl.sh`)

Add this script to your branch and source it before running the project:

```bash
#!/bin/bash
# SYCL Environment Setup for IrisLime
export DUMMY_LIB_PATH="$HOME/src/irislime/dummy_libs"
export LD_LIBRARY_PATH="$DUMMY_LIB_PATH:$LD_LIBRARY_PATH"
export ZES_ENABLE_SYSMAN=1

# Disable tracing to prevent secondary runtime errors
export ZET_ENABLE_LAYER_L0_TRACING=0
export ZET_ENABLE_API_TRACING_LAYER=0

```
