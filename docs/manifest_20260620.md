# IrisLime Research Manifest

## The Objective
To evolve `irislime` into a **self-documenting, forensic-capable research lab.**

## Core Principles
1.  **Idempotency:** The environment should be recreatable from scratch via `make setup` and `make build`.
2.  **Forensic Integrity:** Every build, test, and run must produce a timestamped log (`YYYYMMDD_HMS`) for future regression analysis.
3.  **Out-of-Tree (OOT) Decoupling:** Engine source code (`llama.cpp`) remains pristine; all build artifacts and logic live in the `build/` root.
4.  **Versioned Baselines:** Configuration files (Makefile, config_env) are monitored via a scrub/verify pipeline to prevent configuration drift.
