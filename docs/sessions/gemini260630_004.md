## 2. Bypassing High-RAM Demands on Low-Resource Free Runners

The previous local build failed because compiling high-density SYCL Flash Attention templates simultaneously required over 32 GB of memory. Standard free GitHub-hosted runners are restricted to modest hardware footprints (typically 2 CPU cores and 7 GB of RAM). Trying to run `make -j` on a cloud runner will trigger an instant Out-of-Memory (OOM) kernel panic, killing your workflow process.

To run builds successfully on the free cloud tier without paying for premium high-RAM instances, we inject target-specific parallelism constraints directly into the module files.

#### The Safe Cloud Build Pattern (`.github/workflows/build-matrix.yml`)

Instead of allowing the compiler to thrash, the workflow overrides global concurrency flags, dropping to single-threaded compilation (`-j1`) exclusively for the device-template directories while utilizing full multi-core speeds (`-j2`) for the rest of the application framework. This allows the build to finish cleanly on free infrastructure with zero out-of-pocket cost.

```yaml
# Simplified visual logic map of the budget-gated cloud pipeline
name: IrisLime Automated Budget Build
on: [push]

jobs:
  compile-matrix:
    runs-on: ubuntu-latest
    steps:
      - name: Mount Source & Submodules
        uses: actions/checkout@v4
        with:
          submodules: 'true'

      - name: Mount Local Compiler Cache Node
        uses: actions/cache@v4
        with:
          path: ~/.cache/ccache
          key: ${{ runner.os }}-ccache-${{ github.sha }}
          restore-keys: ${{ runner.os }}-ccache-

      - name: Execute Budget Gated Build
        run: |
          source config_env
          # Force QUIET mode to keep runner logs clean
          # Force RAM safety constraints to fit inside the free 7GB runner block
          make build-sycl QUIET=1 NUM_BUILD_JOBS=1

```
