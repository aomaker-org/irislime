## 1. Objective
Unify the cross-platform execution experience by mapping the legacy `make` muscle memory directly to the modern Python automation layer (`tools/build_runner.py`). This eliminates workflow divergence between the WSL2 Ubuntu bash ecosystem and the native Windows 11 host-side MSVC developer environment.

---

## 2. Target Design Requirements

### 2.1 The `make build` Proxy Target
The top-level `Makefile` must be updated to include a clean, dependency-free `build` rule. When a user executes `make build` on any platform (WSL2 or Native Host with Scoop/MinGW Make), the orchestration engine must transparently route the payload down to the active virtual environment's abstraction script:

```makefile
.PHONY: build
build:
	uv run tools/build_runner.py
